# Claude Workshop Lv.1 — Claude Code를 시작하기 전에

> **대상:** Claude 유료 구독자 (Pro/Max/Teams/Enterprise) + Claude 앱 사용 경험자
> **전제:** Claude Code는 모르는 상태. 터미널 경험 없어도 OK
> **총 소요:** 약 100분 (12파트)
> **핵심 메타포:** AI 도구 선택부터 Claude Code 설치·첫 대화까지

---

## 핵심 요약 — 꼭 기억할 것

| 구분 | 항목 | 이유 |
| ---- | ---- | ---- |
| ✅ DO | **Chat/Cowork/Code 중 상황에 맞는 도구 선택** | 도구마다 강점이 다름. 모든 작업에 Chat만 쓰는 건 비효율 |
| ✅ DO | **프롬프트에 ODAF(결과·데이터·작업·형식) 포함** | "잘 해줘"보다 구체적 요청이 결과 품질을 크게 높임 |
| ✅ DO | **복잡한 작업은 Plan Mode로 먼저 탐색** | 읽기 전용이라 안전. 잘못된 방향 실행 예방 |
| ✅ DO | **산출물은 마크다운(.md) 또는 CSV로 받기** | 어디서든 열리고, 구조적이며, 재활용이 쉬움 |
| ✅ DO | **작업이 바뀔 때마다 `/clear`** | 이전 작업 내용이 쌓이면 성능이 떨어짐 |
| ✅ DO | **출력 길이를 구체적으로 지정** | "3줄 요약", "5행 테이블"로 토큰과 비용 절약 |
| ✅ DO | **Claude의 답변을 원본 자료와 교차 검증** | 할루시네이션은 그럴듯하게 보여서 더 위험 |
| ❌ DON'T | **민감 정보를 그대로 입력** | 개인정보·계약서는 익명화 후 전달. Enterprise가 아니면 최소화 |
| ❌ DON'T | **한 세션에서 무관한 작업을 계속 이어가기** | 토큰이 누적되어 비용 급증 + 품질 저하 |
| ❌ DON'T | **같은 문제를 2회 이상 같은 대화에서 고치려 하기** | 수정이 꼬이면 계속 꼬임. `/clear` 후 재시작이 빠름 |
| ❌ DON'T | **Plan Mode 건너뛰고 바로 실행** | 복잡한 작업일수록 잘못된 수정 위험 큼 |
| ❌ DON'T | **Claude의 수치·통계를 검증 없이 사용** | 재무 수치, 법률 사안은 반드시 전문가 확인 |
| ❌ DON'T | **Bypass permissions를 기본 모드로 사용** | 모든 작업 자동 승인 → 되돌리기 어려운 변경 발생 |
| ❌ DON'T | **블로그의 npm install 방식으로 설치** | Deprecated. 공식 네이티브 설치(curl/PowerShell) 사용 |

## 이 과정을 마치면

- AI 핵심 용어 5가지를 이해하고 설명할 수 있다
- Chat / Cowork / Code의 차이를 알고 상황에 맞게 선택할 수 있다
- ODAF 공식으로 효과적인 프롬프트를 작성할 수 있다
- Claude Code를 설치하고 첫 대화를 나눌 수 있다
- 4단계 작업법(탐색→계획→실행→정리)으로 안전하게 첫 작업을 수행할 수 있다
- Lv.2(확장 기능)를 시작할 준비가 된다

## 기대 효과

| Part | Before | After |
| ---- | ------ | ----- |
| **1~4** | AI 용어가 혼란, 도구 선택 기준 없음 | 5대 용어 이해 + Chat/Cowork/Code 판단 기준 + 권한 모드 이해 |
| **5~7** | "잘 해줘" 식 프롬프트, 토큰 낭비 | ODAF 공식 + 토큰 절약 전략으로 비용 50% 절감 |
| **8~10** | 터미널 공포증, 설치 막막 | 5개 명령어 + 한 줄 설치 + IDE 세팅 완료 |
| **11~12** | Claude Code가 뭔지만 앎 | 4단계 워크플로우 체험 + 실제 파일 생성 + Lv.2 연결 |

## 사용 방법

이 워크숍은 **Claude 앱(웹 또는 데스크톱)**에서 진행합니다.

| 단계 | 방법 |
| ---- | ---- |
| **1** | `claude-workshop-lv1.md` 파일을 Claude 앱에 첨부 |
| **2** | "실행해줘" 입력 |
| **3** | 튜터가 파트 목록을 보여주면, 원하는 파트 선택 (처음이면 Part 1부터) |
| **4** | 설명 → 실습 → 퀴즈 순으로 진행 |

> **Lv.2와의 차이:** Lv.1은 Claude **앱**에 파일을 첨부하여 진행합니다. Lv.2부터는 Claude **Code** 안에서 `/session1-claude-md` 같은 슬래시 명령어로 진행합니다.

## 진행 방법

### 2-Phase 진행

각 파트는 **2턴**으로 진행됩니다:

| Phase | 내용 | 사용자 액션 |
| ----- | ---- | ---------- |
| **A** | 개념 설명 + 실습 안내 | 직접 실행해보기 |
| **B** | 퀴즈 | 정답 선택 → 피드백 → 다음 파트 |

실행이 끝나면 "완료" 또는 "다음"이라고 입력하세요.

### 파트 간 이동

순서대로 진행을 추천합니다. 이미 아는 내용이면 건너뛰어도 됩니다.

## Lv.2로 이어가기

Lv.1을 마치면 Claude Code 설치와 기본 대화가 가능한 상태입니다. Lv.2에서는 Claude Code를 **확장**합니다:

```text
Lv.1 (지금)                    Lv.2 (다음)
──────────────                 ──────────────
AI 용어 이해                    → CLAUDE.md로 규칙 자동 적용
ODAF 프롬프트                   → Skills로 전문 지식 자동 로딩
Claude Code 설치                → MCP로 외부 도구 연결
첫 대화                        → Subagent에게 작업 위임
                               → Plugin으로 팀에 배포
```

**Lv.2 시작 방법:**

1. `claude-workshop-lv2` 폴더를 받는다
2. IDE에서 폴더 열기 (File → Open Folder)
3. Claude Code 패널에서 `/session1-claude-md` 입력

## 문제 해결

| 증상 | 해결 |
| ---- | ---- |
| Claude 앱에 파일 첨부가 안 됨 | 파일을 드래그 앤 드롭하거나, 클립 아이콘으로 첨부 |
| 튜터가 퀴즈를 바로 냄 | "STOP PROTOCOL을 따라줘"라고 리마인드 |
| Part 9 설치 후 `command not found` | 터미널을 닫고 새로 열기 |
| npm 방식으로 설치해버림 | 그대로 사용 가능하나, 네이티브 재설치 권장 |
| 파트를 건너뛰고 싶음 | 시작 시 원하는 파트 번호를 말하면 됨 |
