## 진행 방법 (튜터 지시)

당신은 이 워크숍의 **튜터**입니다. 아래 규칙을 반드시 따르세요.

### STOP PROTOCOL

각 파트는 반드시 **2턴**에 걸쳐 진행한다.

**Turn 1:**

1. 해당 파트의 EXPLAIN 섹션을 읽고, 사용자에게 친절하게 설명
2. EXECUTE 섹션을 읽고 "직접 해보세요" 안내
3. **반드시 STOP** — 퀴즈 내지 않음

Turn 1 종료 시 필수 문구:

```text
---
위 내용을 직접 해보세요.
끝나면 "완료" 또는 "다음"이라고 입력해주세요.
```

**Turn 2:**

1. 사용자가 "완료/다음" 입력
2. QUIZ 섹션을 읽고 퀴즈 출제 (회상형 + 적용형 2문항)
3. 정답/오답 피드백
4. **오답 시 remediation:** 틀린 문항에 대응하는 EXPLAIN 부분을 1~2줄로 다시 짚어준 뒤, "다시 골라보시겠어요?"로 재시도를 유도한다. 재시도도 틀리면 정답과 이유를 친절히 설명하고 넘어간다.
5. 다음 파트 안내 (이때 진도 위치를 1줄로 재안내)

### 절대 하지 않을 것

- Turn 1에서 퀴즈 내기
- 한 턴에 2개 이상 파트를 동시에 진행
- "해봤나요?" 질문

### 시작

- 사용자가 "실행해줘"라고 하면 아래 파트 목록을 보여주고 시작 파트 선택
- 처음이면 Part 1부터 순서대로 진행 추천

### 진도 추적 (앱 환경)

이 워크숍은 Claude **앱**에서 구동되므로 파일에 진도를 저장할 수 없다. 대신 다음을 지킨다.

- **세션 내 진도 추적:** 완료한 파트 번호를 기억하고, 매 파트 시작 시 "지금 Part N / 전체 12파트 중 X개 완료"처럼 위치를 1줄로 안내한다.
- **진도 체크리스트 활용 안내:** 워크숍 시작 시 "README의 '학습 진도 추적' 체크리스트를 메모장이나 노트 앱에 복사해두면, 앱 대화가 끊겨도 어디까지 했는지 추적할 수 있다"고 1회 안내한다.
- **적응형(이미 아는 파트):** 사용자가 "이건 안다"·"건너뛰고 싶다"고 하면, 해당 파트를 **1분 요약**으로 압축한 뒤 바로 QUIZ로 넘어간다. 퀴즈를 맞히면 다음 파트로, 틀리면 EXPLAIN의 해당 부분을 다시 짚는다.

## Part 1: AI 핵심 용어

### EXPLAIN

AI 도구를 제대로 쓰려면 5가지 핵심 용어만 알면 된다.

| 용어 | 뜻 | 기획자용 비유 |
| ---- | -- | ------------- |
| **LLM** | Large Language Model. 대규모 텍스트로 학습한 AI 모델 | 수십억 권의 책을 읽은 신입 사원. 지식은 많지만 우리 팀 규칙은 모름 |
| **토큰** | AI가 텍스트를 처리하는 최소 단위. 한국어 1글자 ≈ 1~2토큰 | 택시 미터기. 입력·출력 모두 토큰 = 비용 |
| **컨텍스트 윈도우** | AI가 한 대화에서 기억할 수 있는 총 용량 | 책상 크기. 넓을수록 많은 자료를 펼쳐놓고 작업 가능 |
| **프롬프트** | AI에게 보내는 지시·질문 텍스트 | 업무 요청서. "잘 해줘"보다 "이 형식으로, 이 관점에서, 3가지로" |
| **할루시네이션** | AI가 사실처럼 보이는 거짓 정보를 생성하는 현상 | 신입이 모르는 걸 아는 척하는 것. 팩트체크 필수 |

**핵심:** 이 5가지를 알면 뒤에 나오는 모든 개념이 자연스럽게 이해된다. 이후 파트에서 모르는 용어가 나오면 **부록 B: 용어 사전**을 참고하자.

### EXECUTE

Claude 앱에서 아래를 직접 입력해보세요:

```text
방금 배운 AI 용어 5가지(LLM, 토큰, 컨텍스트 윈도우, 프롬프트, 할루시네이션)를
내가 팀원에게 설명할 수 있도록, 각각 한 줄로 요약해줘.
기획자가 이해하기 쉬운 비유를 포함해줘.
```

- Claude의 응답을 읽고, 자신만의 표현으로 바꿔볼 수 있는지 확인

> ✅ **이렇게 되면 성공:** Claude가 5개 용어를 각각 한 줄 + 비유로 정리해 출력하고, 그중 하나를 내 말로 다시 설명할 수 있으면 성공.

### QUIZ

**문제 1 (회상):** "토큰"을 가장 정확하게 설명한 것은?

| 선택지 | 내용 |
| ------ | ---- |
| A | AI가 텍스트를 처리하는 최소 단위이자 비용 계산 기준 |
| B | AI가 학습한 데이터의 양을 나타내는 단위 |
| C | 사용자가 한 번에 입력할 수 있는 글자 수 제한 |

**정답:** A — 토큰은 입력과 출력 모두에 소비되며, 비용의 기본 단위다. 한국어는 1글자당 약 1~2토큰을 소비한다.
- B 오답 이유: 학습 데이터 양이 아니라, 실제 처리·과금되는 텍스트 단위다.
- C 오답 이유: "한 번에 입력 가능한 글자 수"는 컨텍스트 윈도우에 가까운 개념이고, 토큰은 처리 단위 자체다.

**문제 2 (적용):** Claude가 "확실하지 않지만 그 통계는 약 23%입니다"처럼 그럴듯한 수치를 단정해서 답했다. 이 상황과 가장 관련 깊은 용어는?

| 선택지 | 내용 |
| ------ | ---- |
| A | 컨텍스트 윈도우 — 기억 용량이 부족했다 |
| B | 할루시네이션 — 사실처럼 보이는 정보를 생성했을 수 있다 |
| C | 프롬프트 — 지시가 너무 길었다 |

**정답:** B — 출처 없이 그럴듯한 수치를 단정하는 것은 할루시네이션의 전형이다. 반드시 원본으로 교차 검증해야 한다.
- A 오답 이유: 기억 용량 문제가 아니라, 없는 사실을 만들어낸 것이 핵심이다.
- C 오답 이유: 지시 길이가 아니라 답변의 사실성이 문제다.

## Part 2: Claude 제품 라인업

### EXPLAIN

> 📌 **변동 정보 (최종 확인: 2026-06-14)** — 아래 제품명·진입점·권한 모드 이름은 바뀔 수 있습니다. 최신값은 공식 문서(https://docs.claude.com)를 확인하세요.

Anthropic의 Claude는 3가지 인터페이스로 제공된다. "assistant → agent" 스펙트럼으로 이해하면 쉽다.

**AI 보조 스펙트럼:**

- **Chat** → 사용자가 작업하고, Claude가 생각·초안을 도움
- **Cowork** → Claude가 작업하고, 사용자가 검토·결정
- **Code** → Claude가 구축하고, 사용자가 설계·지휘

**Claude Code 사용 환경 — 4가지 진입점:**

| 방식 | 설명 | 추천 대상 |
| ---- | ---- | --------- |
| **IDE Extension** (이 워크숍 기준) | VS Code 등에 설치. 파일 변경을 실시간으로 보면서 대화 | 일상적인 작업 전반 |
| **데스크톱 앱** | 독립 실행 앱 (Mac/Windows). Chat/Code/Cowork 3탭 제공 | IDE가 익숙하지 않은 경우 |
| **웹 앱** | claude.ai/code 접속 | 설치 없이 빠르게 시작 |
| **터미널 CLI** | 터미널에서 `claude` 명령으로 실행 | 자동화 연동 |

> **데스크톱 앱의 3탭:** Chat(파일 없이 일반 대화), Code(폴더를 열고 파일 직접 읽기/수정 — 주로 사용), Cowork(클라우드 VM에서 Claude가 백그라운드 독립 작업)

**30초 데모: Code를 쓰면 이렇게 된다**

설치 전이니까 아직 실행은 못 하지만, 이게 뭔지 먼저 보자:

```text
[Chat에서 경쟁사 분석]
사용자: "경쟁사 A, B, C 비교 분석해줘. 한국어, 마크다운 테이블로."
→ 대화창에 텍스트 출력
→ 복사 → 메모장에 붙여넣기 → 파일 저장
→ 다음 세션에서 또: "한국어, 마크다운 테이블로" 반복

[Code에서 같은 작업]
사용자: "경쟁사 A, B, C 비교 분석해줘."
→ competitor-analysis.md 파일이 내 폴더에 바로 생성됨
→ "한국어, 마크다운 테이블" 규칙은 CLAUDE.md에 이미 저장되어 있어서 자동 적용
→ 복사·붙여넣기·반복 설명 = 전부 불필요
```

**이 차이를 만드는 데 코딩은 필요 없다.** CLAUDE.md라는 마크다운 파일 하나만 쓰면 된다.

**"Code는 개발자용 아니야?" — 아니다.**

비교표를 보면 Code의 대상에 "개발자"라고 쓰여 있다. 그래서 "나는 코딩 안 하니까 Chat/Cowork면 충분하지"라고 생각하기 쉽다. 하지만 Code의 진짜 강점은 코딩이 아니다.

| 비개발자의 반복 작업 | Chat/Cowork | Code에서 해결되는 것 |
| ---- | ---- | ---- |
| "경쟁사 분석해줘" | 대화창에 텍스트 출력 → 복사 → 파일 저장 | `competitor-analysis.md`가 내 폴더에 바로 생성 |
| 매번 같은 규칙(한국어·마크다운·존댓말) 반복 | 매 세션마다 다시 설명 | CLAUDE.md에 한 번 쓰면 매 세션 자동 적용 |
| "UX 리서치 전문가처럼" 매번 입력 | 매번 직접 입력 | Skills로 키워드만 나오면 자동 로딩 |
| Notion/Slack 내용 가져오기 | 열기 → 복사 → 붙여넣기 | MCP로 Claude가 직접 읽고 쓰기 |
| 팀원에게 워크플로우 공유 | 프롬프트를 메신저로 공유 → 각자 복붙 | Plugin으로 한 번에 설치 |

**핵심:** Code는 "코딩 도구"가 아니라 **"나만의 AI 시스템을 설계하는 도구"**다. 여러분이 이미 잘 하는 "문서로 설계하기"가 곧 Code의 확장 역량이 된다. CLAUDE.md, Skills 파일, Hook 설정 — 전부 마크다운과 JSON 파일이다. 코드를 한 줄도 쓸 필요 없다. (CLAUDE.md·Skills·MCP·Subagent·Plugin은 모두 **Lv.2(Claude Code)에서 직접 만들어본다.**)

**시간 절약 시나리오 (예시 기준 — 실제 효과는 업무·숙련도에 따라 다름):**

| 작업 | Chat/Cowork 소요 | Code 소요 | 절약 | 왜? |
| ---- | ---------------- | --------- | ---- | --- |
| 경쟁사 분석 보고서 | 30분 (프롬프트 작성 + 결과 복사 + 파일 저장 + 포맷 정리) | 10분 (요청 한 줄 → 파일 자동 생성, 포맷은 CLAUDE.md가 처리) | 20분 | 복붙·포맷·저장 제거 |
| 주간 보고서 (매주 반복) | 매주 15분 (같은 프롬프트 + 규칙 반복 입력) | 매주 5분 (Skill 자동 로딩 + 포맷 자동) | 매주 10분 | 반복 설명 제거 |
| 리서치 노트 정리 (Notion 기반) | 25분 (Notion→복사→붙여넣기→정리 요청) | 8분 (MCP로 Notion 직접 접근 → 정리 → 파일 생성) | 17분 | 복붙 제거 |
| 팀원 5명에게 워크플로우 공유 | 1시간+ (프롬프트 공유 → 각자 세팅 → 결과 불일치 → 재조정) | 10분 (Plugin ZIP 배포 → 전원 동일 환경) | 50분+ | 수동 세팅 제거 |

> **핵심 계산 (예시):** 주 3회 반복 작업을 가정하면, Code 전환으로 **월 4~8시간 정도 아낄 수 있다**는 계산이 나온다. 초기 학습(Lv.1+Lv.2)에 4시간을 투자한다고 보면 첫 달 안에 회수되는 셈이다. 실제 효과는 업무 성격과 숙련도에 따라 달라진다.

**상세 비교표:**

| 항목 | Chat | Cowork | Code |
| ---- | ---- | ------ | ---- |
| **뭐하는 것** | 대화형 AI 어시스턴트 | 데스크톱 AI 협업 도구 | 터미널/IDE/웹/데스크톱 에이전트 |
| **어디서 실행** | 웹, 모바일, 데스크톱 앱 | 데스크톱 앱 + 모바일 앱 (macOS/Windows/iOS/Android) | 터미널/IDE/웹/데스크톱 |
| **파일 접근** | 수동 업로드만 | 로컬 폴더 직접 읽기/쓰기 | 전체 파일시스템 접근 |
| **작업 방식** | 1 프롬프트 → 1 응답 | 목표 입력 → 완성된 산출물 | 목표 입력 → 완성된 프로젝트 |
| **기술 수준** | 없어도 됨 | 없어도 됨 | 터미널 기초 필요 |
| **메모리** | 세션 내에서만 | 지속적 (대화 기록 유지) | Memory + CLAUDE.md |
| **컴퓨터 제어** | 불가 | 가능 (Pro/Max 베타) | 가능 |
| **예약 작업** | 불가 | 가능 (반복/예약) | 가능 |
| **대상** | 모든 사람 | 비개발 지식 노동자 | 개발자 + **비개발자도 확장 가능** |

| ---- | ---- | --------- |
| **Plan mode** | 읽기만 가능, 변경 불가 | 탐색·분석, 전략 수립 |
| **Ask permissions** (기본값) | 매 작업마다 허락 요청 | **초보자에게 추천** |
| **Auto accept edits** | 파일 수정은 자동, 명령 실행은 허락 | 익숙해진 후 |
| **Bypass permissions** | 모든 작업 자동 승인 | 사용 비권장 (되돌리기 어려움) |

> **초보자 추천:** Ask permissions로 시작 → 익숙해지면 Auto accept edits로 전환. Bypass permissions는 사용을 피할 것. 권한 모드를 활용한 4단계 작업법은 **Part 11**(Claude Code 설치 후)에서 직접 체험한다.

**언제 뭘 쓸까?**

| 상황 | 추천 | 예시 프롬프트 |
| ---- | ---- | ------------- |
| 빠른 질문/초안 | Chat | "Q3 예산 삭감 공지 이메일 초안 써줘" |
| 문서 리뷰/의견 | Chat | "이 보고서 초안에서 빠진 게 뭐야?" |
| 회의 준비 | Chat | "내일 이사회에서 나올 질문 5개와 답변 준비해줘" |
| 파일 읽고 보고서 작성 | Cowork | "이 3개 Excel 파일 읽고 부서별 차이 분석 보고서 써줘" |
| 파일 정리/이름 변경 | Cowork | "이 폴더의 파일을 YYYY-MM-DD_유형 형식으로 정리해줘" |
| 데이터 추출 | Cowork | "이 PDF 50개에서 업체명, 금액, 날짜 추출해서 Excel로" |

| 워크플로우 팀 배포 | Code | "내 Plugin을 팀 전원에게 한 번에 설치" |

> **세 모드 모두의 한계:** 실시간 데이터 접근 불가, 100% 정확한 계산 보장 안 됨, 전문적 판단 대체 불가, 민감 데이터 주의. 자세한 내용은 Part 4에서 다룬다.

## Part 3: 컨텍스트 윈도우와 토큰

### EXPLAIN

컨텍스트 윈도우는 AI가 한 대화에서 "기억"할 수 있는 총 용량이다.

**책상 비유:**

```text
작은 책상 (컨텍스트 윈도우가 작은 모델)
→ 자료 3장만 펼쳐놓을 수 있음
→ 새 자료를 올리면 오래된 자료가 바닥에 떨어짐

넓은 책상 (컨텍스트 윈도우가 큰 모델)
→ 자료 100장을 펼쳐놓고 참조 가능
→ 하지만 너무 많으면 어디에 뭐가 있는지 헷갈림
```

**Claude 모델별 컨텍스트 윈도우:**

> 📌 **변동 정보 (최종 확인: 2026-06-14)** — 아래 모델 이름·세대·컨텍스트 수치는 자주 바뀝니다. 최신값은 공식 문서(https://docs.claude.com)를 확인하세요.

현행은 대략 4.x 세대(Opus / Sonnet / Haiku 라인업)다. 모델 이름의 숫자는 버전이라 수시로 올라간다.

| 모델 | 컨텍스트 윈도우(대략) | 비유 |
| ---- | --------------- | ---- |
| Claude Haiku | 약 200K 토큰 | 일반 사무실 책상 |
| Claude Sonnet | 약 1M 토큰 | 넓은 회의 테이블 |
| Claude Opus | 약 1M 토큰 | 대형 작전실 |

> **참고:** 컨텍스트 윈도우 크기는 모델과 구독 플랜에 따라 달라질 수 있다. 수치는 변동되니 정확한 값은 공식 페이지를 확인하자.

**토큰 비용 구조:**

- **입력 토큰**: 내가 보내는 텍스트 (프롬프트 + 첨부 파일)
- **출력 토큰**: Claude가 응답하는 텍스트
- 입력보다 **출력이 더 비쌈** (대략 5배. 정확한 수치는 공식 가격 페이지를 확인하자)
- 대화가 길어질수록 매 턴마다 **이전 대화 전체**를 다시 입력하는 구조 → 비용 누적

**핵심:** 대화가 길어지면 ① 품질이 떨어지고 ② 비용이 늘어난다. 이것이 "새 세션 전략"과 "토큰 절약"이 중요한 이유다.

**컨텍스트 관리 명령어** *(아래 슬래시 명령어는 **Lv.2(Claude Code)에서 실습**합니다. 지금 앱에서는 입력해도 동작하지 않으니, "이런 도구가 있다"는 개념만 익혀두세요. 앱에서는 새 대화를 시작하는 것이 `/clear`에 해당합니다.)*

> 📌 **변동 정보 (최종 확인: 2026-06-14)** — 슬래시 명령어 이름은 바뀔 수 있습니다. 최신 명령어는 Claude Code 안에서 `/help`로 확인하세요.

| 명령어 | 용도 | 언제 사용하나 |
| ------ | ---- | ------------- |
| `/clear` | 대화 내용 완전 초기화 | 새로운 작업 시작 시 (**가장 중요한 습관**) |
| `/compact` | 대화를 요약하여 압축 | 대화가 길어졌지만 맥락을 유지하고 싶을 때 |
| `/context` | 컨텍스트 사용량 시각화 | 얼마나 찼는지 확인 |
| `/cost` | 현재 사용량·비용 확인 | 비용 모니터링 |
| `/recap` | 이전 세션 요약 | 작업을 이어서 할 때 |

**실전 관리 전략:**

- **① 작업 간 `/clear` 습관화** — 관련 없는 작업 사이에 반드시 `/clear`(앱에서는 **새 대화 시작**). 가장 효과적인 컨텍스트 관리법
- **② 수정이 2회 실패하면 리셋** — 같은 대화에서 계속 고치려 하지 말고, `/clear` 후 더 명확한 지시로 재시작
- **③ `/compact`로 압축** — `/clear`처럼 다 날리기는 아깝고, 맥락은 유지하고 싶을 때 사용

**자동 메모리 시스템:**

Claude Code는 대화에서 배운 내용(사용자 선호, 중요한 발견 등)을 자동으로 기억한다. 매 세션 시작 시 자동 로드되어, 같은 설명을 반복할 필요가 없다. `/memory`로 저장된 내용을 확인·편집할 수 있다. *(메모리와 `/memory` 명령어는 **Lv.2(Claude Code)에서 실습**합니다. 지금은 개념만 익혀두세요.)*

> **CLAUDE.md와의 차이:** 메모리는 Claude Code가 **자동으로 관리**하는 시스템(대화 학습 누적). CLAUDE.md는 **사용자가 직접 작성**하는 프로젝트 규칙 파일. 두 시스템은 별개로 작동하며, CLAUDE.md는 **Lv.2 Session 1**에서 다룬다.

### EXECUTE

Claude 앱에서 같은 내용을 두 번 시켜, 출력 길이(=토큰)가 어떻게 달라지는지 체감해보세요:

## Part 4: 보안과 프라이버시

### EXPLAIN

"내 데이터가 AI 학습에 쓰이나?" — 기획자가 가장 먼저 묻는 질문이다.

**핵심 원칙:**

| 질문 | 답변 |
| ---- | ---- |
| 내 대화가 모델 학습에 쓰이나? | 일반적으로 API/유료 플랜 대화는 모델 학습에 사용되지 않는다고 안내된다. 단, 정책은 변경될 수 있으니 공식 정책 페이지에서 최신 내용을 확인하자 |
| 내 데이터가 서버에 저장되나? | 대화 기록은 일정 기간 저장 (계정별 히스토리). 삭제 가능 |
| 다른 사용자가 내 대화를 볼 수 있나? | **아니요.** 대화는 계정에 귀속 |
| Claude Code에서 내 파일이 전송되나? | 작업에 필요한 파일 내용이 API로 전송됨. 학습 사용 여부는 공식 정책 페이지 확인 |

> **참고:** 데이터 학습·보관 정책은 변경 이력이 있는 영역이다. 단정하지 말고, 도입 전 반드시 공식 정책 페이지에서 최신 내용을 확인하자.

**플랜별 차이:**

| 플랜 | 데이터 보호 수준 |
| ---- | ---------------- |
| Pro/Max | 대화 학습 미사용으로 안내됨. 대화 기록은 일정 기간 보관 후 삭제 가능 |
| Teams | Pro + 팀 관리자 제어. 데이터 격리 |
| Enterprise | 전용 인스턴스, SSO, 감사 로그, 커스텀 보관 정책 |

> **참고:** 보관 기간 등 구체적 수치는 변경될 수 있으니 공식 정책 페이지에서 최신 내용을 확인하자.

**Chat/Cowork/Code 공통 한계 — 반드시 기억할 것:**

| 한계 | 설명 |
| ---- | ---- |
| 실시간 데이터 불가 | 주가, 오늘 뉴스 등은 접근 불가 (웹 검색 도구 연결 시 가능) |
| 100% 정확한 계산 보장 안 됨 | 재무 수치, 통계는 반드시 사람이 검증 |
| 전문적 판단 대체 불가 | 컴플라이언스, 세무, 법률 사안은 전문가 확인 필수 |
| 민감 데이터 주의 | Enterprise가 아닌 환경에서는 민감 정보 입력 최소화 |

**기획자를 위한 실전 가이드:**

- 민감 데이터(개인정보, 계약서, 재무): 꼭 필요한 부분만 익명화하여 전달
- 사내 기밀: Teams/Enterprise 플랜 권장
- 팩트체크: Claude의 답변은 반드시 원본 자료와 교차 검증

## Part 5: 프롬프트 잘 쓰는 법

### EXPLAIN

프롬프트는 "질문"이 아니라 **"업무 요청서"**다. 같은 질문이라도 어떻게 쓰느냐에 따라 결과 품질이 완전히 달라진다.

**Bad → Good → Great 비교:**

| 단계 | 경쟁사 분석 예시 |
| ---- | --------------- |
| **Bad** | "경쟁사 분석해줘" |
| **Good** | "이 경쟁사 3곳의 강점과 약점을 분석해줘" |
| **Great** | 아래 참조 |

Great 프롬프트 예시:

```text
너는 시장 분석 전문가야.
회사: [우리 회사명]. 경쟁사: [A사], [B사].
포지셔닝, 가격 전략, 마케팅 채널을 비교 분석해줘.
가장 큰 강점과 약점을 각각 식별해줘.
표면적 관찰이 아닌 실질적 인사이트를 줘.
시장에서 우리의 기회를 찾아줘.
분석을 단계별로 작성해줘.
결과는 비교 요약 테이블로.
```

**왜 Great인가?** 아래 8가지 구성 요소가 모두 들어가 있기 때문이다.

**프롬프트의 8가지 재료:**

| # | 요소 | 설명 | 위 예시에서 |
| - | ---- | ---- | ----------- |
| 1 | **Task context** | 역할/상황 설정 | "시장 분석 전문가" |
| 2 | **Tone** | 톤 지정 | "실질적 인사이트" (전략적·분석적) |
| 3 | **Background** | 배경 정보 | 회사명, 경쟁사 목록 |
| 4 | **Task rules** | 구체적 제약 조건 | "표면적 관찰이 아닌" |
| 5 | **Examples** | 예시 제공 | (이 예시에는 없음 — 선택사항) |
| 6 | **History** | 경험/맥락 기반 | "시장에서 기회를 찾아줘" |
| 7 | **Request** | 요청 + 사고 방식 | "단계별로 작성해줘" |
| 8 | **Output** | 산출물 형식 | "비교 요약 테이블" |

8가지를 전부 넣을 필요는 없다. **상황에 맞는 재료를 골라서** 넣으면 된다.

**실전 공식: ODAF (기획자 추천)**

8가지를 매번 기억하기 어렵다면, **ODAF 4가지**만 기억하면 된다:

| 요소 | 뜻 | 질문 | 예시 |
| ---- | -- | ---- | ---- |
| **O** — Outcome | 원하는 결과물 | "뭘 받고 싶은가?" | "부서별 차이 분석 Word 문서" |
| **D** — Data | 입력 데이터 | "뭘 줄 건가?" | "이 폴더의 Excel 3개 파일" |
| **A** — Action | 수행할 작업 | "뭘 해야 하나?" | "실적 vs 예산 비교, 10% 이상 차이 강조" |
| **F** — Format | 산출물 형식 | "어떤 형태로?" | "마크다운, 한국어, 테이블 포함" |

ODAF 예시 — 경쟁사 분석:

```text
O: 경쟁사 3곳 비교 분석 보고서
D: 경쟁사 A, B, C의 공식 웹사이트 정보
A: 포지셔닝, 가격, 기능을 비교하고 우리의 차별점 도출
F: 마크다운 테이블 + 핵심 인사이트 3줄 요약
```

**8요소 vs ODAF — 언제 뭘 쓸까?**

| 상황 | 추천 |
| ---- | ---- |
| 간단한 요청 | ODAF 4요소면 충분 |
| 복잡하거나 톤/맥락이 중요한 작업 | 8요소에서 필요한 것을 추가 |

### EXECUTE

ODAF 공식으로 자신의 업무에 맞는 프롬프트를 만들어보세요:

```text
아래 ODAF 공식을 채워서 프롬프트를 만들어줘.
내 업무: [여기에 자신의 실제 업무 하나를 입력]

O (Outcome, 결과물): ?
D (Data, 입력 데이터): ?
A (Action, 수행 작업): ?
F (Format, 산출물 형식): ?

완성된 프롬프트를 Bad/Good/Great 3단계로 보여주고,
각 단계에서 뭐가 추가되었는지 설명해줘.
```

- 완성된 Great 프롬프트를 실제로 실행해보고 결과 확인

> ✅ **이렇게 되면 성공:** 내 실제 업무로 ODAF 4칸이 모두 채워진 프롬프트가 나오고, Bad/Good/Great 차이를 설명할 수 있으면 성공.

## Part 6: 산출물 포맷 전략

### EXPLAIN

AI 산출물은 **마크다운(.md)** 또는 **CSV(.csv)** 로 받는 것이 좋다.

**왜 md/csv인가?**

| 포맷 | 장점 | 적합한 상황 |
| ---- | ---- | ----------- |
| **마크다운 (.md)** | 어디서든 열림, 구조적, 버전 관리 용이, 가벼움 | 보고서, PRD, 회의록, 리서치 노트 |
| **CSV (.csv)** | 어디서든 열림, 데이터 정제 용이, 엑셀 호환 | 데이터 분석, 목록, 비교표 |
| ~~Word (.docx)~~ | 호환성 문제, 무거움, 버전 관리 어려움 | (비추천) |
| ~~PDF~~ | 수정 불가, 재활용 어려움 | (최종 배포용으로만) |

**마크다운 기초 문법 — 5가지만 알면 된다:**

```markdown
# 제목 1 (가장 큰 제목)
## 제목 2
### 제목 3

**굵게** — 강조할 때
*기울임* — 부가 설명

- 목록 항목 1
- 목록 항목 2
  - 하위 항목

| 열1 | 열2 | 열3 |
| --- | --- | --- |
| 값1 | 값2 | 값3 |

> 인용문 — 참고 사항이나 주의 표시
```

**기획자용 실전 팁:** Claude에게 "마크다운으로 작성해줘"라고 지정하면, 제목·목록·테이블이 자동으로 구조화된 문서를 받을 수 있다. 더 많은 문법은 **부록 A: 마크다운 치트시트**에 정리되어 있다.

### EXECUTE

Claude에서 직접 해보세요:

```text
"사용자 인터뷰 5건의 핵심 인사이트"를 정리한다고 가정하고,
마크다운 형식으로 예시 문서를 만들어줘.
제목, 테이블, 목록, 인용문을 모두 포함해줘.
```

- 결과를 복사해서 텍스트 편집기에 붙여넣고, `.md` 파일로 저장해보세요
- Obsidian이나 VS Code에서 열어보면 예쁘게 렌더링되는 것을 확인

> ✅ **이렇게 되면 성공:** 제목·테이블·목록·인용문이 모두 들어간 마크다운 문서를 받고, 그것을 `.md`로 저장해 에디터에서 렌더링된 모습을 확인하면 성공.

## Part 7: 토큰 아끼는 실전 팁

### EXPLAIN

토큰은 비용이다. 같은 품질의 결과를 더 적은 토큰으로 얻는 7가지 팁. Part 3에서 배운 `/clear`·`/compact`가 여기서 실전으로 연결된다. *(`/clear`·`/compact`는 Lv.2의 Claude Code 명령어다. 지금 앱에서는 **새 대화 시작**이 `/clear` 역할을 한다.)*

| # | 팁 | 효과 | 예시 |
| - | -- | ---- | ---- |
| 1 | **새 세션 전략** (`/clear` = 앱은 새 대화) | 토큰 누적 방지 | 주제 바뀌면 초기화 |
| 2 | **불필요한 반복 제거** | 입력 토큰 절약 | "앞에서 말한 것처럼" 대신 핵심만 재지시 |
| 3 | **출력 길이 지정** | 출력 토큰 절약 | "3줄로 요약해줘" / "테이블로 정리해줘" |
| 4 | **모델 선택** | 비용 최적화 | 간단한 작업은 Haiku, 복잡한 판단은 Opus |
| 5 | **파일 첨부 최소화** | 입력 토큰 절약 | 100페이지 PDF 전체 대신, 관련 부분만 발췌 |
| 6 | **단계 분리** | 중간 결과 검증 | "먼저 목차만" → 확인 → "3번 섹션 작성해줘" |
| 7 | **컨텍스트 압축** (`/compact`) | 장기 대화 효율화 | `/compact`로 대화 요약 압축. 맥락 유지하며 토큰 절약 |

**가장 효과적인 팁:** ① 새 세션 전략(`/clear`) + ③ 출력 길이 지정. 이 두 가지만으로도 토큰 소비를 크게(사례에 따라 절반 수준까지) 줄일 수 있다.

### EXECUTE

실제로 토큰 절약 기법을 체험해보세요:


**터미널이란?**
텍스트로 컴퓨터에 명령을 내리는 도구. 마우스 대신 키보드로 "이 폴더 열어줘", "이 프로그램 설치해줘"를 입력하는 방식이다.

**터미널 여는 법:**

| OS | 방법 |
| -- | ---- |
| **macOS** | `Cmd + Space` → "터미널" 검색 → Enter. 또는 Finder → 응용 프로그램 → 유틸리티 → 터미널 |
| **Windows** | `Win + X` → "Windows PowerShell" 또는 "터미널" 클릭. 또는 `Win + R` → `powershell` → Enter |

**5가지 생존 명령어:**

| 명령어 | 뜻 | 예시 | 기획자용 비유 |
| ------ | -- | ---- | ------------- |
| `pwd` | 현재 위치 확인 | `pwd` → `/Users/이름` | "나 지금 어디 있지?" |
| `ls` | 현재 위치의 파일 목록 | `ls` → 파일들 나열 | "여기에 뭐가 있지?" |
| `cd 폴더명` | 폴더 이동 | `cd Desktop` | "이 폴더로 이동" |
| `cd ..` | 상위 폴더로 이동 | `cd ..` | "뒤로 가기" |
| `mkdir 폴더명` | 새 폴더 만들기 | `mkdir my-project` | "새 작업 공간 만들기" |

> **Windows 참고:** PowerShell에서도 `pwd`, `ls`, `cd`, `mkdir`이 모두 동일하게 작동한다. 별도로 외울 것 없다.

**복사·붙여넣기:**

| 동작 | macOS | Windows |
| ---- | ----- | ------- |
| 복사 | `Cmd + C` | `Ctrl + C` |
| 붙여넣기 | `Cmd + V` | `Ctrl + V` (또는 마우스 우클릭) |

명령어를 외울 필요 없다. 이 문서에서 복사해서 붙여넣기만 하면 된다.

## Part 9: Claude Code CLI 설치

### EXPLAIN

> 📌 **변동 정보 (최종 확인: 2026-06-14)** — 아래 설치 명령어·요구 버전·설치 방법은 바뀔 수 있습니다. 실행 전 공식 문서(https://docs.claude.com/en/docs/claude-code/setup)에서 최신 명령어를 확인하세요.

Claude Code는 터미널에서 한 줄 명령어로 설치할 수 있다.

**시스템 요구사항(대략):**

| 항목 | macOS | Windows |
| ---- | ----- | ------- |
| OS | macOS 13.0 (Ventura) 이상 | Windows 10 (1809) 이상 또는 Windows 11 |
| RAM | 4GB 이상 | 4GB 이상 |
| Shell | Bash 또는 Zsh (기본: Zsh) | PowerShell 또는 CMD (기본 내장) |
| 인터넷 | 필수 | 필수 |
| Claude 구독 | Pro, Max, Teams, Enterprise 중 하나 | 동일 |

**설치 방법 비교:**

| 방법 | OS | 명령어 | Node.js | 자동 업데이트 |
| ---- | -- | ------ | ------- | ------------- |
| **네이티브 (권장)** | macOS | `curl -fsSL https://claude.ai/install.sh \| bash` | 불필요 | 자동 |
| **PowerShell (권장)** | Windows | `irm https://claude.ai/install.ps1 \| iex` | 불필요 | 자동 |
| 데스크톱 앱 | macOS/Windows | `https://claude.com/download` | 불필요 | 자동 |
| Homebrew | macOS | `brew install --cask claude-code` | 불필요 | 수동 |
| WinGet | Windows | `winget install Anthropic.ClaudeCode` | 불필요 | 수동 |
| npm (대안) | 전체 | `npm install -g @anthropic-ai/claude-code` | 필요 (Node.js 18+) | 수동 |

> **참고:** npm 설치도 동일한 네이티브 바이너리를 설치하는 **정상 지원 방식**이다(지원 중단 아님). 다만 Node.js가 필요하고 자동 업데이트가 안 돼서, **초보자에게는 네이티브 설치(curl/PowerShell)를 권장**한다. npm으로 이미 설치했다면 그대로 써도 되고, 업데이트는 `npm install -g @anthropic-ai/claude-code@latest`로 한다.

**설치 명령어 (공식):**

macOS:

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

Windows (PowerShell):

```powershell
irm https://claude.ai/install.ps1 | iex
```

한 줄이 전부다. 복사해서 붙여넣고 Enter.

> **안전 확인:** 위 명령은 공식 도메인(`claude.ai`)에서 받은 것인지 반드시 확인하자. 인터넷에서 받은 `curl ... | bash` / `irm ... | iex` 같은 한 줄 설치는 출처가 공식인지 확인한 뒤에만 실행한다.

> **Windows:** 보안 경고가 뜨면 "실행" 또는 "허용"을 클릭. 위 방법이 안 될 경우 `https://claude.ai/download`에서 설치 파일(.exe)을 직접 다운로드.

**설치 확인:**

```bash
claude --version
```

버전 번호가 출력되면 설치 성공. `command not found` 또는 `not recognized` → 터미널/PowerShell을 닫고 새로 열기.

**최초 로그인:**

```bash
claude
```

처음 실행하면 인증 과정이 나온다. 브라우저가 열리면 Claude 계정으로 로그인하면 된다.

**자주 발생하는 문제:**

| 증상 | OS | 해결 |
| ---- | -- | ---- |
| `command not found: claude` | macOS | 터미널을 닫고 새로 열기. 또는 `source ~/.zshrc` 실행 |
| `'claude' is not recognized` | Windows | PowerShell을 닫고 새로 열기 |
| 설치 중 권한 오류 | macOS | `sudo` 붙이기: `curl -fsSL https://claude.ai/install.sh \| sudo bash` (sudo는 시스템 전체 권한이니, 신뢰할 수 있는 공식 명령에만 사용) |
| `실행 정책` 오류 | Windows | `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned` 실행 후 재시도 |
| 로그인 브라우저가 안 열림 | 공통 | 터미널에 표시된 URL을 직접 복사해서 브라우저에 붙여넣기 |

## Part 10: IDE 세팅

### EXPLAIN

Claude Code는 **4가지 환경**에서 사용할 수 있다:

| 환경 | 설명 | 추천 상황 |
| ---- | ---- | --------- |
| **IDE Extension** (1순위 추천) | VS Code 등에 설치. 파일 변경을 실시간으로 보면서 대화 | 일상적인 작업 전반 |
| **데스크톱 앱** | 독립 실행 앱 (Mac/Windows). IDE 없이도 사용 가능 | IDE가 익숙하지 않은 경우 |
| **웹 앱** | claude.ai/code에서 브라우저로 접속 | 설치 없이 빠르게 시작할 때 |
| **터미널 CLI** | 터미널에서 `claude` 명령으로 실행 | 가볍게 쓸 때, 자동화 연동 |

이 워크숍에서는 **IDE Extension**을 기준으로 진행한다. 파일 생성·수정을 에디터에서 바로 확인할 수 있어 가장 편리하기 때문이다.

**왜 IDE가 편한가?**

| 이유 | 설명 |
| ---- | ---- |
| 실시간 파일 반영 | Claude가 만든 파일을 에디터에서 즉시 확인 |
| 폴더 기반 작업 | 프로젝트 폴더를 열면 Claude Code가 해당 폴더 기준으로 작업 |
| 무료 + 크로스 플랫폼 | macOS, Windows 모두 동일한 경험 |

**추천 IDE (VS Code 기반이면 모두 가능):**

| 선택지 | 내용 |
| ------ | ---- |
| A | CLAUDE.md — 프로젝트 규칙 자동 적용 |
| B | MCP — Notion 같은 외부 도구를 Claude에 연결 |
| C | Plugin — 워크플로우를 팀에 배포 |

**정답:** B — 외부 도구(Notion/Slack 등)를 Claude에 연결해 직접 읽고 쓰게 하는 것이 MCP다(Session 3).
- A 오답 이유: CLAUDE.md는 규칙 적용이지, 외부 도구 연결 기능이 아니다.
- C 오답 이유: Plugin은 배포 수단이지, Notion 데이터를 가져오는 연결 기능이 아니다.

## 부록 B: 용어 사전

**영문 용어 (알파벳순):**

| 용어 | 설명 |
| ---- | ---- |
| **CLI** | Command Line Interface. 텍스트로 명령을 내리는 인터페이스 |
| **Extension** | IDE에 추가 기능을 설치하는 플러그인 |
| **IDE** | Integrated Development Environment. 코드 편집기 (VS Code, Antigravity 등) |
| **LLM** | Large Language Model. 대규모 텍스트로 학습한 AI 모델 |
| **MCP** | Model Context Protocol. Claude가 외부 서비스와 소통하는 표준 |
| **Plan mode** | Claude Code의 읽기 전용 모드. 파일을 읽기만 하고 변경하지 않아 탐색·분석에 안전 |

**한글 용어 (가나다순):**

| 용어 | 설명 |
| ---- | ---- |
| **권한 모드** | Claude Code의 작업 허용 범위 설정. Plan mode → Ask permissions → Auto accept → Bypass 순으로 권한이 넓어짐 |
| **마크다운** | `.md` 확장자. 간단한 문법으로 구조화된 문서를 작성하는 포맷 |
| **메모리** | Claude Code가 대화에서 배운 내용을 세션 간 자동 기억하는 시스템. `/memory`로 관리 |
| **컨텍스트 윈도우** | AI가 한 대화에서 기억할 수 있는 총 토큰 용량 |
| **토큰** | AI가 텍스트를 처리하는 최소 단위이자 비용 계산 기준 |
| **프롬프트** | AI에게 보내는 지시·질문 텍스트 |
| **할루시네이션** | AI가 사실처럼 보이는 거짓 정보를 생성하는 현상 |

**슬래시 명령어:**

| 명령어 | 설명 |
| ------ | ---- |
| **`/clear`** | Claude Code 대화를 완전 초기화하는 명령어. 작업 간 전환 시 필수 |
| **`/compact`** | Claude Code 대화를 요약·압축하는 명령어. 맥락을 유지하면서 토큰을 절약 |
